## 제품 한 줄
환자는 평소처럼 AI와 일상 대화만 한다. 시스템은 그 대화에서 수면·약물·기분·스트레스원·위험 신호를 조용히 추출해, 의사가 진료 전 10초 안에 파악할 수 있는 요약 리포트로 정리한다. **진단·치료는 하지 않는다 — 기록·요약·전달 도구.**

## 설계 원칙 (Non-negotiable)
1. **진단·치료 안 함.** 의료법상 "기록·요약·전달" 도구. 진단명 단정·약물 변경 지시 금지.
2. **위험 감지는 규칙 기반(프론트 `RISK_WORDS`)으로 항상 동작.** LLM/API에 의존하지 않음 — 안전 기능은 API가 죽어도 작동.
3. **Solar(Upstage) API 키는 프론트에 절대 안 둠.** Render 백엔드 프록시가 대신 호출, 키는 Render 환경변수(`UPSTAGE_API_KEY`)에만.
4. **데모는 목업 폴백으로 안 깨짐.** `BACKEND_URL`이 비었거나 호출 실패 시 규칙 기반 목업으로 동작.
5. **의사 추가 업무 0.** 의사는 기록하지 않고 소비만. 리포트는 예외(Red Flag)만 부각.
6. **위험 신호 시** 섣부른 위로로 덮지 않고, 따뜻하게 위기 자원(자살예방상담 109) 안내 + 의사에게 Red Flag.

## 아키텍처
```
[환자 채팅 UI] ──┐
                 │  (BACKEND_URL 비면 목업 모드)
[규칙 기반 추출]  ├─→ getSignals(text)
[규칙 기반 위험감지]│        │ BACKEND_URL 있으면 fetch
                 │        ▼
                 │   [Render 프록시 /extract] ──→ [Solar(Upstage) API]
                 │        │ (5초 타임아웃 / 실패 시)
                 └────────┴─→ 목업 폴백
                          ▼
                 [의사 대시보드 — 진료 전 요약 리포트]
```
- **프론트(`index.html`)**: 단일 파일. 환자 채팅 + 의사 대시보드 + 데모 재생.
- **백엔드(`backend/server.js`)**: Express. `/extract`(Solar 호출)·`/`(헬스체크). 키는 환경변수에서만 읽음.
- **배포**: 프론트 = GitHub Pages, 백엔드 = Render(Free).

- MindHub는 대용량 배경 영상이나 복잡한 스크롤 캔버스를 복제하지 않고, 심사자가 기능 데모를 빠르게 찾을 수 있는 기존 구조를 유지했다.
- 첫 화면을 중앙 집중형 대형 타이포그래피와 전체 폭 제품 미리보기로 정리해 핵심 가치가 먼저 읽히도록 했다.
- “일상 대화 → 임상 신호 → 진료 전 요약”을 한 화면에서 설명하는 스토리 구간을 추가했다.
- `IntersectionObserver` 기반의 가벼운 등장 효과를 적용하고 `prefers-reduced-motion` 환경에서는 모션을 제거했다.
- 320px 모바일과 일반·다크 모드에서 가로 스크롤, 문장 줄바꿈, 제품 미리보기 겹침을 점검했다.

## 2026-06-20 — 상담봇형 장문 응답 제거

- 실제 모바일 사용 화면에서 `뭐해`, `배고파`, `오늘 비 와?` 같은 짧은 말에도 설명·조언·후속 질문을 길게 붙이는 문제를 확인했다.
- 물음표가 있다는 이유만으로 모든 문장을 작업 요청으로 분류하던 규칙을 제거했다.
- 자주 쓰는 짧은 일상말과 짧은 감정 표현은 Solar를 호출하지 않고 1~2문장의 자연스러운 직접 응답으로 처리해 속도와 토큰 사용을 함께 줄였다.
- 위기 표현에는 AI가 연락처와 번호 목록을 반복하지 않고 현재 혼자인지·즉각적 계획이나 준비가 있는지만 확인하도록 했다. 연락처는 별도 안전 카드가 담당한다.
- 직전 위기 대화 뒤 무기력·고립 표현이 이어지면 일반 생활 조언으로 빠지지 않고 짧은 안전 후속 확인을 유지한다.
- 모델이 지시를 벗어나 번호 목록을 생성하면 비작업형 답변에서 제거하고, 토큰 제한으로 끝난 답변은 마지막 완성 문장까지만 표시한다.
- 첫 인사, AI 아바타, 하단 고지와 안전 카드 문구도 상담기관형 표현을 줄이고 간결하게 정리했다.

## 2026-06-20 — 새 대화 시작 화면 간소화

- 앱을 열었을 때 AI가 먼저 말풍선으로 말을 거는 방식을 제거했다.
- 대화가 없는 화면 중앙에 작은 심볼과 `어떤 이야기부터 해볼까요, 이름` 문구만 표시한다.
- 저장된 대화가 있는 사용자가 새 대화를 열면 `다시 이야기해 볼까요, 이름`으로 자연스럽게 바뀐다.
- 사용자가 입력을 시작하거나 기존 대화를 열면 중앙 안내가 즉시 사라진다.
- 안내 문구는 대화 기록에 저장되지 않으며 AI API도 호출하지 않는다.
