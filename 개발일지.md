# 개발일지 — 마음기록 (MindHub MVP)

> 전국민 AI 리부트 활용대회 출품작 · 1인 팀 · 최종 발표 2026-06-22
> 퇴원 정신과 환자의 일상 AI 대화에서 임상 신호를 뽑아 **의사용 진료 전 요약 리포트 1장**으로 전달하는 데모.

---

## 제품 한 줄
환자는 평소처럼 AI와 일상 대화만 한다. 시스템은 그 대화에서 수면·약물·기분·스트레스원·위험 신호를 조용히 추출해, 의사가 진료 전 10초 안에 파악할 수 있는 요약 리포트로 정리한다. **진단·치료는 하지 않는다 — 기록·요약·전달 도구.**

## 설계 원칙 (Non-negotiable)
1. **진단·치료 안 함.** 의료법상 "기록·요약·전달" 도구. 진단명 단정·약물 변경 지시 금지.
2. **위험 감지는 규칙 기반(프론트 `RISK_WORDS`)으로 항상 동작.** LLM/API에 의존하지 않음 — 안전 기능은 API가 죽어도 작동.
3. **Solar(Upstage) API 키는 프론트에 절대 안 둠.** Render 백엔드 프록시가 대신 호출, 키는 Render 환경변수(`UPSTAGE_API_KEY`)에만.
4. **데모는 목업 폴백으로 안 깨짐.** `BACKEND_URL`이 비었거나 호출 실패 시 규칙 기반 목업으로 동작.
5. **의사 추가 업무 0.** 의사는 기록하지 않고 소비만. 리포트는 예외(Red Flag)만 부각.
6. **위험 신호 시** 섣부른 위로로 덮지 않고, 따뜻하게 위기 자원(자살예방상담 109) 안내 + 의사에게 Red Flag.

## 아키텍처
```
[환자 채팅 UI] ──┐
                 │  (BACKEND_URL 비면 목업 모드)
[규칙 기반 추출]  ├─→ getSignals(text)
[규칙 기반 위험감지]│        │ BACKEND_URL 있으면 fetch
                 │        ▼
                 │   [Render 프록시 /extract] ──→ [Solar(Upstage) API]
                 │        │ (5초 타임아웃 / 실패 시)
                 └────────┴─→ 목업 폴백
                          ▼
                 [의사 대시보드 — 진료 전 요약 리포트]
```
- **프론트(`index.html`)**: 단일 파일. 환자 채팅 + 의사 대시보드 + 데모 재생.
- **백엔드(`backend/server.js`)**: Express. `/extract`(Solar 호출)·`/`(헬스체크). 키는 환경변수에서만 읽음.
- **배포**: 프론트 = GitHub Pages, 백엔드 = Render(Free).

## 개발 로그

### 2026-06-19 — 초기 MVP
- 프론트 단일 파일(`index.html`) 구현: 환자 채팅 UI, 규칙 기반 신호 추출(`extractSignals`), 위험 감지(`RISK_WORDS`), 의사 리포트(①수면·기능 ②약물 순응도 ③안전·자살 리스크 ④스트레스원 ⑤정신상태) + Red Flag 트리아지.
- "데모 시나리오 재생"(가상 환자 3일치: 불면 → 무응답 → 자살 사고)으로 무입력 시연 가능.
- Render용 Solar 프록시(`backend/server.js`) 작성. 키는 `UPSTAGE_API_KEY` 환경변수로만.
- 위험 감지(`red_flag`/`evidence`)는 Solar 결과와 무관하게 **항상 클라이언트 규칙 기반 유지** — 안전을 API에 의존시키지 않음.
- 호출 실패 시 목업 폴백으로 데모 안 깨지게 설계.

### 2026-06-19 — 데모 견고화 (Render cold start 대비)
- `getSignals()`의 백엔드 `fetch`에 `AbortController` 기반 **5초 타임아웃** 추가.
- 초과 시(예: Render 무료 티어 cold start ~30초) abort → 기존 `catch` 폴백 경로를 그대로 타 **즉시 목업으로 전환**. `finally`에서 타이머 정리.
- `RISK_WORDS`(규칙 기반 위험 감지)와 목업 폴백 로직은 **변경 없음**.

### 2026-06-19 — 백그라운드 임상 추출 연결 (작업순서 4)
- `app.html`에 세션 누적 신호 저장소 `collected=[]` 추가 — 대화 뒤에서 모은 신호를 리포트(작업순서 6)에서 사용할 자리.
- `collectSignals(text)` 추가: 기존 `getSignals()`(Solar `/extract` + 5초 타임아웃 + 규칙 기반 red_flag 보존 + 목업 폴백)를 감싸 `{at, text, sig}`로 누적. `getSignals`가 자체 try/catch로 폴백하므로 throw 없음 — 안전·대화 기능에 영향 0.
- `sendMsg`: 막아뒀던 추출 호출을 실연결. **`await` 없이 병렬 호출**(fire-and-forget) → 답변 지연 0, 화면에 추출 표시 없음(비가시).
- `newChat`: 새 대화 시작 시 `collected=[]`로 초기화.
- 원칙 준수: 비가시성(개발용 표시 없음) · 위험 감지는 규칙 기반으로 네트워크 무관하게 항상 기록 · 5초 타임아웃·목업 폴백 유지 · 키 프론트 미노출.
- **다음 단계(미구현):** 위기 자원 인라인 카드(5) · 리포트 화면(6, `collected` 사용) · 설정(7).

### 2026-06-19 — 리포트 화면 + 환자 비공개 선택 (작업순서 6)
- `app.html`에 풀스크린 리포트 오버레이(⑤) 추가. 계정 메뉴 "리포트 기록 보기"/"내보내기" → `openReport()`.
- `aggregate()`로 `collected`(작업순서 4 누적)를 집계 → 5영역: ① 수면·기능(평균+막대 sparkline) ② 약물 순응도(복약 n/m일) ③ 안전·자살 리스크 ④ 스트레스원·환경 ⑤ 정신상태·기타(기분 평균). 로직은 `index.html`의 `makeReport` 구조 재사용.
- **환자 통제권:** 항목별 공유/비공개 토글(`reportShare`, `secRow`/`toggleShare`). 단 **안전(③ Red Flag)은 잠금 🔒 항상 포함**(빼기 불가 — 온보딩 고지됨). red_flag 있으면 맨 위 트리아지 카드 + 근거 원문.
- **의사에게 보여주기(전체화면):** `enterDoctorView()` — 환자 통제 UI(`share-btn`)·비공개 항목·하단 액션 숨김. `rvBack()`로 환자 화면 복귀.
- **PDF 내보내기:** `exportPDF()`=`window.print()`. `@media print`로 앱 크롬·비공개 항목 제외하고 리포트만 인쇄(→PDF 저장).
- disclaimer: "진단이 아니라 대화 기록 요약, 의학 판단은 선생님이". `newChat`에서 `reportShare` 기본값 복원.
- 원칙 준수: 진단 단정 없음 · 안전 항목 강제 포함 · 의사 추가 업무 0(예외만 부각·10초 파악) · 데모 가상 데이터.
- **다음 단계(미구현):** 설정(7, 외래 일정·담당 병원 연락처(`hospitalContact`)·처방 약물·공유 기본값·온보딩 다시 보기).

- 프론트 두 경로로 서빙됨: **GitHub Pages**(`forblune.github.io/mindhub-mvp/`, 푸시 시 자동배포) + **Render**(`mindhub-mvp.onrender.com`, server.js 정적 서빙).
- Render Auto-Deploy를 **On Commit**으로 설정(이후 푸시마다 자동 재배포). 단, 설정 켜기 전 이미 푸시된 커밋은 자동배포 대상이 아니라 **Manual Deploy → Deploy latest commit**으로 1회 따라잡아야 함.
- 발표 안정성: 프론트 확인·시연은 GitHub Pages가 더 빠르고 안정적. Render는 `/chat`·`/extract` 백엔드가 필요할 때 사용.

## 현재 상태
- ✅ 프론트 목업 모드 동작 (`index.html`, GitHub Pages 배포).
- ✅ v2 챗봇형 앱(`app.html`) — 작업순서 **1~7 전부 기능**(UI 골격 · `/chat` 실연결 · 백그라운드 추출 · 위기 카드 · 리포트·비공개 선택 · 설정). 실배포 연동·발표 리허설만 남음.
- ✅ 백엔드 Solar 프록시 코드 완성 + 5초 타임아웃 폴백.
- ⬜ 실배포 연동(C~F): Upstage 키·모델 확보 → Render 배포 → `BACKEND_URL` 연결 → 실추출 테스트. *(키/콘솔 작업 필요)*
- ⬜ (선택) Supabase 저장·개인 도메인 — 발표 필수 아님.

## 발표 대비
- Render 무료 티어 cold start 대비: 발표 직전 백엔드 주소 한 번 열어 깨워두기.
- 불안하면 발표는 **목업 모드(`BACKEND_URL` 비움)** — 절대 안 깨짐.
- 데모 데이터는 전부 **가상 환자**. 실제 개인 진료내용 넣지 않음.
