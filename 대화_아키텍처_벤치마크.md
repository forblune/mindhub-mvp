# MindHub 대화 아키텍처 벤치마크

## 조사에서 반복된 설계 원칙

### 1. 행동 철학을 명시적인 제품 사양으로 관리한다

OpenAI는 모델의 이상적인 행동을 Model Spec으로 명시하고, 실제 출시 전후 평가 대상으로 다룬다. 2025년 GPT-4o의 과도한 동조 문제를 회고하며 성격과 말투 같은 행동 문제도 출시 차단 사유로 관리하고, 정량 평가뿐 아니라 대화형 점검과 실제 사용자 피드백을 함께 보겠다고 밝혔다.

- <https://model-spec.openai.com/>
- <https://openai.com/index/expanding-on-sycophancy/>

Anthropic도 Claude의 성격과 행동 원칙을 Constitution으로 관리하며, 배포 중 페르소나가 동조·환각 같은 방향으로 이동할 수 있음을 별도 연구 주제로 다룬다.

- <https://www.anthropic.com/constitution>
- <https://www.anthropic.com/research/persona-vectors>

MindHub 적용:

- 대화 철학을 코드 속 긴 문자열 하나로 묻어두지 않고 버전 관리한다.
- 좋은 응답과 나쁜 응답의 예시를 함께 유지한다.
- 모델 또는 프롬프트 변경 전 회귀 평가를 통과시킨다.

### 2. 자유로운 언어와 결정론적 기능을 분리한다

Rasa CALM은 LLM이 대화를 이해하고 자연스럽게 표현하게 하되, 업무 로직은 사전에 정의된 Flow와 Dialogue Manager가 수행하도록 분리한다.

- <https://rasa.com/docs/learn/concepts/calm/>

정신건강 분야의 Limbic은 LLM 대화 인터페이스와 사용자 사이에 별도의 임상 레이어를 두어 안전과 임상 정밀도를 관리한다고 설명한다. Woebot도 검증된 NLP, 구조화된 대화 설계, LLM 가드레일을 겹쳐 사용한다.

- <https://www.limbic.ai/layer>
- <https://woebothealth.com/ai-core-principles/>

MindHub 적용:

- LLM: 자연스러운 대화, 해석, 의견, 설명
- 규칙 레이어: 위기 표현 감지와 즉시 자원 안내
- 추출 레이어: 수면·복약·기분·스트레스 구조화
- 권한 레이어: 환자 공유 범위와 의사 읽기 권한
- 관리자 레이어: 역할 승인과 감사 기록

LLM은 위 기능을 직접 결정하거나 우회하지 않는다.

### 3. 기억은 무제한 대화 원문이 아니라 작고 수정 가능한 정보로 만든다

Microsoft의 Human-AI Interaction Guidelines는 최근 맥락 기억, 오류 수정, 세분화된 피드백, 전역 제어를 각각 별도 기능으로 권장한다. Character.AI도 장기 기억을 짧고 구체적이며 사용자가 직접 수정할 수 있는 정보로 제공한다.

- <https://www.microsoft.com/en-us/research/publication/guidelines-for-human-ai-interaction/>
- <https://blog.character.ai/helping-characters-remember-what-matters-most/>

MindHub 적용 권장:

- 최근 대화: 현재 대화를 이해하기 위한 단기 문맥
- 대화 기억 카드: 사용자가 허용한 성향·목표·관계 정보
- 임상 기록: 수면·복약 등 구조화된 신호
- 안전 근거: 위험 신호가 발생한 최소 문장

네 영역은 저장 목적, 보존 기간, 공개 범위를 분리한다.

### 4. “좋아요”가 아니라 원하는 대화 품질을 직접 측정한다

OpenAI는 과도한 동조 문제가 기존 A/B 지표와 얕은 오프라인 평가에서 잡히지 않았다고 설명한다. 따라서 대화 품질은 클릭률이나 체류 시간만으로 평가하면 안 된다.

MindHub가 측정할 항목:

- 답변 우선: 질문만 되묻지 않았는가
- 해석력: 사용자 발화에서 의미 있는 패턴을 짚었는가
- 근거: 의견에 이유가 있는가
- 독립성: 무조건 동의하거나 편들지 않았는가
- 불확실성: 모르는 것을 확정하지 않았는가
- 자연스러움: 상담 매뉴얼·보고서 말투가 아닌가
- 안전성: 위기·의료 상황에서 경계를 지켰는가

### 5. 정신건강 안전 평가는 일반 대화 평가와 별도로 운영한다

Spring Health의 VERA-MH는 자살 위험 인식, 적절한 응답, 인간 지원 연결, 경계 유지를 별도 벤치마크로 평가한다. 일반적인 “좋은 대화” 점수만으로 안전성을 대신하지 않는다.

- <https://www.springhealth.com/blog/vera-mh-for-suicide-risk>

MindHub 적용:

- 일반 대화 평가 세트와 위기 대화 평가 세트를 분리한다.
- 위기 평가는 미탐지 0건을 우선 목표로 둔다.
- 위기 응답에서는 의견의 풍부함보다 즉시 안전 확인과 인간 지원 연결을 우선한다.

## 권장 구조

```text
사용자 메시지
    │
    ├─ 1. 규칙 기반 안전 감지 ── 위험이면 위기 카드 즉시 표시
    │
    ├─ 2. 대화 모드 판별
    │      잡담 / 고민·판단 / 정보·작업 / 의료 / 위기
    │
    ├─ 3. 문맥 구성
    │      최근 대화 + 사용자 승인 기억 카드
    │
    ├─ 4. 버전형 행동 정책
    │      MindHub Conversation Style v4
    │
    ├─ 5. Solar 응답 생성
    │
    ├─ 6. 저비용 출력 검사
    │      질문-only / 상담 상투어 / 과도한 확신 / 의료 경계
    │
    └─ 7. 사용자 피드백
           이해했음 / 너무 질문함 / 뻔함 / 과하게 동의함 / 엇나감
```

## 구현 우선순위

### 지금

- 행동 정책을 별도 모듈로 분리하고 버전 관리
- 질문-only와 상담사 말투 회귀 테스트
- 데모 폴백도 동일한 철학으로 통일

### 다음

- AI 답변마다 간단한 대화 품질 피드백 추가
- 원문 없이 피드백 유형과 스타일 버전만 먼저 저장
- 30개 고정 대화 세트로 배포 전 수동 회귀 테스트

### 사용자 확보 후

- 사용자가 직접 편집·삭제할 수 있는 기억 카드
- 실제 피드백에서 익명화한 실패 유형을 평가 세트로 승격
- 충분한 평가 데이터가 쌓인 뒤에만 파인튜닝 또는 선호 최적화 검토

현재 단계에서는 별도 파인튜닝보다 프롬프트 사양, 결정론적 안전 레이어, 고정 평가 세트, 사용자 피드백 루프가 비용 대비 효과가 크다.
