# 21팀 경진대회 구현 평가 확인 요청서

## 먼저 인정할 부분

- 발표 시간이 초과되었고, 짧은 시간 안에 핵심 기능과 기술 구조를 명확히 보여주지 못했습니다.
- 제출된 루트 주소에서는 구형 `index.html` 데모가 먼저 노출되고, 완성도가 높은 환자 앱과 의사 화면은 각각 `app.html`, `doctor.html` 경로에 분리되어 있었습니다.
- 따라서 평가자가 완성본 기능을 한 번에 확인하기 어려웠던 것은 제출자 본인의 동선 설계 책임도 있습니다.

이 요청은 발표 감점을 없애 달라는 요청이 아니라, **구현 평가가 구형 루트 화면만을 기준으로 이루어졌는지 확인해 달라는 요청**입니다.

---

## 확인을 요청하는 이유

### 1. 경진대회 점수와 PBL 강사평가의 차이

경진대회 사전평가:

- 총점: 55.0/100
- 주제: 14.5/20
- 아이디어: 13.7/20
- 사업화 가능성: 12.5/20
- 구현·디자인: 7.0/20
- 구현·프로그래밍: 7.3/20
- 평가 6건, 작성된 종합평 없음

동일 프로젝트의 PBL 기록:

- 총점: 99/100
- 문제 인식 강사평가: 15/15
- 문제 정의 강사평가: 10/10
- 문제 분해 강사평가: 10/10
- 추상화 강사평가: 20/20
- 알고리즘 설계 강사평가: 19/20
- 구현 강사평가: 15/15
- 결과 시험·발표 강사평가: 9/10

발표 전달력에 부족함이 있었다는 점은 PBL의 9/10과도 일치합니다. 다만 구현 두 항목의 7점대 평가는 실제 구현 범위와 차이가 커, 완성본 경로가 평가 과정에서 확인되었는지 문의드립니다.

### 2. 평가일 이전 Git 기록으로 확인되는 구현

2026년 6월 19일 Git 기록:

- 11:26 — Solar 기반 범용 `/chat` 실연결
- 11:58 — 대화 속 임상 신호 백그라운드 추출
- 12:01 — 네트워크와 독립된 위기 자원 카드
- 12:05 — 진료 전 리포트와 환자 항목별 비공개 선택
- 12:09 — 다음 진료일·병원·약물·공유 설정
- 12:17 — GitHub Pages와 Render 배포 검증
- 12:28 — 대화 기록 저장·검색·이어가기
- 13:16 — Supabase 클라우드 저장/읽기 활성화
- 14:05 — 환자·의사 로그인 및 역할 분리

이는 단순 화면 목업이 아니라, 평가일에 다음 흐름이 구현되어 있었음을 보여줍니다.

1. 환자 일반 AI 대화
2. Solar를 통한 임상 후보 문장 구조화
3. 수면·복약·기분·스트레스·안전 신호 누적
4. 환자의 항목별 공유 범위 선택
5. Supabase 저장 및 환자/의사 권한 분리
6. 의사의 읽기 전용 진료 전 리포트
7. AI 장애 시 폴백 및 로컬 규칙 기반 안전 안내

### 3. 대회 우대 조건과 실제 Solar 사용

대회 안내에는 국내 LLM인 Solar 활용이 가산점 대상으로 명시되어 있습니다.

본 프로젝트는 Solar를 이름만 언급한 것이 아니라 Render 백엔드의 `/chat`, `/extract` 경로에 실제 연결했습니다. API 키는 프론트엔드가 아니라 서버 환경변수로 관리했습니다.

## 요청 사항

특별 가점이나 발표 감점 취소를 요청드리는 것이 아닙니다.

다만 아래 두 사항을 확인해 주시면 감사하겠습니다.

1. 구현·디자인 및 구현·프로그래밍 평가 시 `app.html`과 `doctor.html` 완성본이 확인되었는지
2. 구형 루트 화면만 확인된 경우, 평가일 이전 Git 이력과 당시 완성본 기능을 기준으로 구현 항목을 재확인할 수 있는지

발표 시간 초과와 완성본 진입 동선을 명확히 만들지 못한 책임은 인정합니다. 그와 별개로 실제 구현 범위가 평가에 반영되었는지만 확인을 부탁드립니다.

## 바로 보낼 수 있는 짧은 메시지

안녕하세요. 21팀 마음기록 프로젝트 평가와 관련해 구현 항목의 확인을 조심스럽게 요청드립니다.

발표 시간이 초과되었고 핵심을 충분히 전달하지 못한 점, 제출 루트에서 구형 데모가 먼저 보이도록 만든 점은 제 책임입니다. 다만 완성본은 평가일 당시 `/app.html`과 `/doctor.html`에 별도로 배포되어 있었고, Git 기록상 6월 19일 12시 전후까지 Solar 대화·임상 신호 추출·환자 공유 통제·진료 전 리포트가, 14시경에는 환자/의사 역할 분리까지 구현되어 있었습니다.

동일 프로젝트의 PBL 강사평가는 알고리즘 19/20, 구현 15/15, 결과·발표 9/10이었으나 경진대회 구현 항목은 디자인 7.0/20, 프로그래밍 7.3/20으로 차이가 커서, 평가 과정에서 완성본 경로가 확인되었는지 문의드립니다.

발표 감점 취소나 특별 대우를 요청드리는 것은 아닙니다. 구형 루트 화면만 확인된 경우 평가일 이전 Git 이력과 당시 완성본을 기준으로 구현 항목을 한 번 재확인해 주실 수 있을지 부탁드립니다.
