# 21팀 MVP — 배포·연동 실행 순서 (GPT/Codex와 함께)

> 현재 상태: `index.html`(프론트, 목업 데모 동작) + `backend/`(Solar 프록시) 코드 완성.
> 아래 순서대로 하면 "데모 → 실제 동작 → 배포"까지 됨. 각 단계 막히면 GPT에 "에러 메시지 + 무엇을 하려는지"를 그대로 붙여 물어보기.

---

## A. GitHub에 올리기
- [ ] VSCode에서 `21팀_MVP` 폴더 열기
- [ ] Source Control(가지 아이콘) → **Publish to GitHub** → 이름 `mindhub-mvp`, **Public**
- [ ] 올라간 주소 확인: `github.com/forblune/mindhub-mvp`
- ⚠️ `.env`·API 키는 절대 커밋하지 않기 (.gitignore에 막아둠)

## B. 프론트 배포 (GitHub Pages) — 제출 링크 확보
- [ ] 저장소 → **Settings 탭**(계정 설정 아님, *저장소* 설정) → 왼쪽 **Pages**
- [ ] Branch `main` / 폴더 `(root)` → **Save**
- [ ] 몇 분 뒤 `https://forblune.github.io/mindhub-mvp/` 접속 확인
- [ ] "데모 시나리오 재생" 눌러 동작 확인 (이 시점엔 목업 모드)

## C. Solar(Upstage) 키·모델 확보
- [ ] console.upstage.ai → **API 키 발급**, 대회 크레딧 적용 확인
- [ ] 콘솔에서 **정확한 모델명** 확인 (solar-mini / solar-pro2 / 등 — 실제 보이는 이름)
- [ ] 키는 메모장 같은 데 잠깐만, 코드엔 절대 안 넣음

## D. 백엔드(Render) 배포 — Solar 프록시 (키 숨김)
- [ ] Render → New → **Web Service** → `mindhub-mvp` 저장소 연결
- [ ] 설정: Root Directory `backend` / Build `npm install` / Start `npm start` / Free
- [ ] **Environment(환경변수):**
  - `UPSTAGE_API_KEY` = (발급한 키)
  - `SOLAR_MODEL` = (콘솔에서 본 모델명)
- [ ] 배포 후 주소 확인 → 예: `https://mindhub-mvp.onrender.com`
- [ ] 브라우저로 그 주소 열어 **`ok`** 뜨면 성공

## E. 프론트 ↔ 백엔드 연결
- [ ] `index.html` 상단 `const BACKEND_URL = "";` 에 Render 주소 입력
  - 예: `const BACKEND_URL = "https://mindhub-mvp.onrender.com";`
- [ ] 저장 → 커밋·푸시 (Pages 자동 재배포)

## F. 테스트
- [ ] 배포된 페이지에서 직접 메시지 입력(예: "어제 4시간밖에 못 잤어, 약은 먹었어")
- [ ] 의사 리포트에 수면·복약이 Solar로 추출돼 들어가는지 확인
- [ ] 위험 문장("다 끝내고 싶어") → Red Flag + 위기 안내 뜨는지 확인
- [ ] Solar 꺼도(BACKEND_URL 비우면) 목업으로 도는지 = 발표 백업 확인

## G. (선택) Supabase 저장
- [ ] supabase.com 프로젝트 → URL + anon key
- [ ] 테이블 `daily_logs`, `reports` 생성 (스키마는 준비물_체크리스트.md 참고)
- [ ] **Row Level Security(RLS)** 켜고 정책 설정
- [ ] 프론트에 supabase-js 추가해 기록 저장/불러오기 (GPT에 "supabase-js insert/select 예제" 요청)

## H. (선택) Cloudflare 개인 도메인 연결
- [ ] GitHub Pages: 저장소 Settings → Pages → Custom domain 입력
- [ ] Cloudflare DNS에 CNAME 추가 (GPT에 "GitHub Pages custom domain Cloudflare CNAME 설정" 요청)

---

## ⚠️ 발표 대비 (꼭)
- Render 무료 티어는 **cold start**(첫 요청 ~30초). 발표 직전 백엔드 주소 한 번 열어 깨워두기.
- 그래도 불안하면 발표는 **목업 모드(BACKEND_URL 비움)**로 — 절대 안 깨짐.
- 데모 데이터는 전부 **가상 환자**. 실제 개인 진료내용 넣지 않기.

## Codex/GPT에 넘길 때 한 줄 맥락
"진단 아님, 기록·요약·전달 도구. 위험 감지는 규칙 기반(프론트)로 항상 동작해야 함. Solar는 Render 프록시로 키 숨김. 데모는 목업 폴백으로 절대 안 깨지게." → 이 원칙 깨는 코드는 거절.
